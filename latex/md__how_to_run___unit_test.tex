\hypertarget{md__how_to_run___unit_test_autotoc_md1}{}\doxysection{\texorpdfstring{1. About the Unit Test}{1. About the Unit Test}}\label{md__how_to_run___unit_test_autotoc_md1}
This project includes a {\bfseries{unit test for the User Client}}, which interacts with a server by sending various commands and verifying the responses. The test ensures that the server and client communication works as expected, particularly focusing on user-\/related commands like\+:


\begin{DoxyItemize}
\item User login and registration.
\item Session handling (getting session ID, logging in).
\item User account management (creating users, retrieving user info).
\item Database creation for users and accounts.
\end{DoxyItemize}

The test is implemented using the {\bfseries{Boost Unit Test Framework}} and is designed to simulate interactions with the server and analyze the received data to ensure correctness.

Each unit test is aimed at testing individual commands, such as {\ttfamily login}, {\ttfamily reg\+\_\+user}, {\ttfamily me}, {\ttfamily user\+\_\+list}, and others. The main purpose of this unit test is to validate the server\textquotesingle{}s response for various client commands, checking both expected success and error messages.

The following core functions are tested\+:
\begin{DoxyItemize}
\item {\bfseries{Login tests}} (valid and invalid login scenarios)
\item {\bfseries{Registration tests}} (new users and existing users)
\item {\bfseries{User information retrieval}}
\item {\bfseries{Account creation and management}}
\end{DoxyItemize}
