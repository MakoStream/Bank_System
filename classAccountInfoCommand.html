Detailed Description

Retrieves detailed information about an account by card number.

Validates card number, checks if account exists, retrieves IBAN, balance, currency, and owner information, logs all actions via logEye.

Note
Requires: <string>, Account, handleInfo, logEye
Syntax: account_info <card_number>

void AccountInfoCommand::execute ( handleInfo &  handle)
inlineoverridevirtual

Executes the account information retrieval command.

Retrieves detailed account information based on a provided card number (PAN). The method:

  • Logs operation start and session details
  • Parses the input command into arguments
  • Validates the number of arguments
  • Validates the card number length (must be 16 digits)
  • Checks whether an account with the given card number exists
  • Fetches the account and retrieves:
    • PAN
    • IBAN
    • Balance
    • Currency type
    • Account owner user ID
  • Returns the formatted account information to the client
  • Sets handle.sessionData.hash[0] to 1 on success (0 on failure)
  • Finalizes logging with SUCCESS or FAILURE status
Parameters
handleReference to handleInfo containing session data, raw command string, and the hash array for execution status.
Exceptions
Throwsa response via throw_response() in case of:
  • Missing arguments
  • Invalid card number length
  • Account with the provided PAN not found
Note
Side effects:
  • Produces trace logs via logEye
  • Modifies handle.sessionData.hash[0]
  • Sends a session/network response through throw_response()
Return values
voidThis method does not return a direct value. Execution result is communicated via handle.sessionData.hash[0]:
  • 0 on failure
  • 1 on successful data retrieval
